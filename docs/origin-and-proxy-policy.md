# Origin and proxy policy

Proxy mode is available only for A, AAAA, and CNAME hostnames and is mutually exclusive with Geo-DNS. Each proxied hostname has exactly one origin containing scheme, host, port, Host header, optional TLS SNI, verification choice, bounded connect/response timeouts, no more than two retries, and an explicit WebSocket choice. Redirects are never followed.

The control plane rejects unspecified, loopback, IPv4-mapped, NAT64 well-known, carrier NAT, link-local, benchmark, multicast, reserved, and cloud-metadata destinations. Administrators manage additional blocked networks/addresses and the narrow private-origin allow-list under **Platform settings → Origin destination safety** (or through the typed settings API/CLI). These values are revisioned in PostgreSQL and are not environment configuration. An allow-list can never permit loopback, metadata, link-local, multicast, or translation-prefix bypasses. The edge resolves again immediately before connecting and applies the same built-in and PostgreSQL-backed CIDR policy to every A and AAAA result, preventing DNS rebinding and mixed safe/unsafe answer sets.

The edge removes hop-by-hop and client-supplied forwarding headers, sets the configured origin Host, and creates trusted forwarding metadata. It rejects conflicting Content-Length/Transfer-Encoding framing, malformed names or values, oversized headers/bodies, unknown Host/SNI, and unapproved upgrades before origin work. HTTP/3 stays disabled. HTTP/2 stream, reset, header-list, and connection request counts must remain explicitly bounded.

Non-apex proxied names reference the shared platform proxy hostname. Apex records use healthy service addresses from the same placement model. Edge health changes update shared routing state; they are not a reason to compile every domain artifact.

Each enabled service-pool hostname is rendered as bounded PowerDNS Lua backed by static per-country, per-continent, and global RRsets containing only registered, enabled, non-drained, listener-ready, heartbeat-fresh cells. Equivalent candidates use deterministic hashed spread. The platform proxy hostname aliases the default shared pool; quarantine and dedicated domains reference their assigned pool and follow the same IPv4/IPv6 readiness rules.
